# on-worldcup2026-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面均为静态网页，彼此之间无依赖关系，可单独访问或部署。

## 项目简介

本项目是一个通用型静态 HTML 页面仓库，用于存储和提供若干独立制作的网页内容。页面内容涵盖信息展示、资料整理等用途，不针对任何特定网站或域名。仓库的主要目的是提供一个集中存放和版本管理 HTML 文件的平台，方便后续查阅、分享或二次利用。

## 目录结构

```
/
├── pages/          # 存放所有独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件代表一个独立页面，文件名即页面标识。
- `assets/`：存放页面共用的样式、脚本、图片等资源文件，按类型分目录管理。

## 页面归档说明

- 所有页面均为纯静态 HTML，无需后端服务即可在浏览器中直接打开。
- 页面之间相互独立，不会互相调用或共享数据。
- 每个页面均包含完整的 HTML 结构，可单独部署至任何静态托管服务。
- 仓库不跟踪页面的外部链接或来源，仅提供文件本身。

## 使用方式

1. **本地查看**：克隆仓库后，直接用浏览器打开 `pages/` 目录下的任意 HTML 文件。
2. **部署到服务器**：将整个仓库或 `pages/` 目录上传至任意静态文件服务器，即可通过对应 URL 访问各页面。
3. **二次开发**：可自由修改、复制或引用仓库中的 HTML 及资源文件，无需额外授权。
